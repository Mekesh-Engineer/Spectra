## 4.1 HARDWARE COMPONENTS

The physical realization of the Spectra inspection system requires a carefully selected suite of hardware elements balanced for cost-efficiency, processing capability, and deployment flexibility. The primary hardware architecture revolves around edge-node internet-of-things (IoT) devices that act as the sensory input for the deeper application layers. This embedded architecture aims for minimal footprint, reducing mechanical obstruction while ensuring rigorous industrial reliability.

The essential hardware components integrated into the final physical prototype include:

- ESP32-CAM microcontroller module with OV2640 sensor
- Two SG90 Micro Servo Motors for pan and tilt configuration
- Lithium-Ion (18650) multi-cell battery system for standalone power
- Battery Holder
- Mini Breadboard and Jumper Wires
- USB-to-Serial Adapter (FTDI)
- 3D Printed custom Camera Holder
- 16Ã—2 LCD Display (Optional)
- Piezo Buzzer (Optional)

## 4.2 HARDWARE COMPONENTS DESCRIPTION

### 4.2.1 ESP32-CAM MODULE

The ESP32-CAM is the core computational and imaging component of the hardware layer. Built around an ESP32 chip with a dual-core 32-bit processor and integrated WiFi capability, this module captures high-resolution imagery. It comes equipped with an OV2640 image sensor, supporting resolutions up to UXGA (1600Ã—1200) and frame rates up to 30 FPS. In this implementation, the resolution is generally scaled to VGA (640Ã—480) to optimize the balance between detection clarity and streaming bandwidth, ensuring real-time MJPEG transmission over the local network to the AI detection backend.

### 4.2.2 SG90 SERVO MOTOR

Camera positioning is governed by two SG90 micro servo motors. These are lightweight, high-torque (approx. 1.8 kg-cm at 4.8V) rotary actuators that allow for precise angular control between 0 and 180 degrees. Operating on a PWM (Pulse Width Modulation) signal generated by the ESP32 microcontroller, one motor is designated for panning (horizontal axis) and the other for tilting (vertical axis). This dynamic freedom allows the system to sweep across a conveyor belt or scanning bed, increasing the effective field of view without requiring multiple camera installations.

### 4.2.3 LITHIUM-ION BATTERY (18650)

Untethered operation is achieved through a dedicated power supply using rechargeable 3.7V Lithium-Ion 18650 battery cells. Known for their high energy density and consistent discharge capabilities, these batteries are essential to support the relatively high peak current demands (often >500mA) triggered when the ESP32 engages its WiFi radio stack simultaneously with servo motor actuation.

### 4.2.4 BATTERY HOLDER

A standardized 2-cell 18650 battery holder is used to secure the cylindrical lithium cells. Designed with internal series or parallel spring contacts and external leads, the holder facilitates secure mechanical retention in high-vibration environments, while safely routing the current to the breadboard's power rails.

### 4.2.5 MINI BREADBOARD

The mini breadboard serves as the fundamental physical matrix for the circuit prototype. It facilitates solderless connections between the power rails, the ESP32-CAM module, servo control lines, and any auxiliary components. The compact size of the mini breadboard allows it to be neatly packaged alongside the base of the camera mount, minimizing wire lengths and reducing electromagnetic interference across the logic signals.

### 4.2.6 USB-TO-SERIAL ADAPTER

Because the ESP32-CAM module lacks a built-in USB interface for programming, an external USB-to-Serial (FTDI) adapter is necessary during the firmware development phase. The adapter bridges the USB protocol from the development workstation to the UART interface of the ESP32, allowing the compiled C/C++ firmware to be flashed to the device's flash memory.

### 4.2.7 CAMERA HOLDER

To structurally unite the ESP32-CAM and the two SG90 servo motors, a rigid, 3D printed camera holder is utilized. This bespoke mechanical chassis is fabricated using PLA or PETG filament, offering a stable pan-tilt bracket. Careful tolerance design ensures the ESP32-CAM friction-fits tightly into the bezel, ensuring optical stability during rotational movements, which is critical for preventing motion-blur during AI frame extraction.

### 4.2.8 16Ã—2 LCD DISPLAY

An optional 16Ã—2 alphanumeric Liquid Crystal Display (LCD) provides local, on-device diagnostic output. When integrated via an I2C communication interface (to conserve the ESP32-CAM's limited GPIO pins), the screen conveys real-time system states, local IP addresses, connection statuses, and immediate inspection tallies, granting operators immediate visual feedback without requiring dashboard access.

### 4.2.9 PIEZO BUZZER

A piezo buzzer supplements the hardware node by offering auditory alerts. Triggered by digital high/low signals from the ESP32, the buzzer sounds distinct tonal sequences to warn operators of error conditions (e.g., WiFi disconnection) or alerts generated by the backend layer (e.g., critical threshold of defective pipes detected).
